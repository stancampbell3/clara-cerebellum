# Typed Edges — Live E2E Example

The exact scenario used to verify the typed-edges phase
(`docs/ritual_typed_edges.md`, "Verification / Live E2E", 2026-07-05):
a two-node Ritual where a deduce-capable Clara node consults a Groq
evaluator over an `offering` edge and synthesizes both answers.

## Files

- `reasoned_response.pl` — the authored Prolog for node `n1`. It calls
  `consult_clara_groq/2`, which is **generated by edge transduction** from
  the `offering` edge `e1` (n1 → n2), not authored — activation merges the
  generated helper with this source.
- `graph_layout.json` — the Cobbler graph: `n1` Clara
  (`clara_mind_splinter`, ollama), `n2` Clara/Groq (`groq_evaluator`), one
  `offering` edge `e1`. `prologSource` is left empty here; `run_e2e.sh`
  injects `reasoned_response.pl` so the .pl file stays the single source
  of truth.
- `run_e2e.sh` — registers a scratch user, focuses `clara_mind_splinter`,
  creates + activates the config on lildaemon (:6666), and runs a query.

## What happens at runtime

1. Run pushes the goal; `n1`'s deduction calls
   `ponder_text_with_context/3` (local LLM answer A), then the generated
   `consult_clara_groq/2` → `caws_consult('n2', 'consults/e1', ...)`.
2. `caws_offer/4` emits an `evaluator/offering` Coire event; the
   CycleController lifts it onto a routed `TephraEnvelope`
   (`target_node_id: n2`, fresh `correlation_id`) and registers a pending
   offer — convergence blocks while any offer is outstanding.
3. `n2`'s RitualParticipant (addressed match) evaluates via Groq and
   replies Hohi echoing the correlation id.
4. `ingest_tephra` resolves the pending offer, writes the reply into both
   engine mailboxes as `ritual/hohi`; `caws_await/2` binds answer B and
   the generated CLIPS hook asserts `edge_replied('e1')`.
5. `n1` synthesizes A + B with a second ponder; the deduction converges.
   On Tabu or patience timeout the consult **fails** (timeout-to-false)
   and convergence still completes.

Dis log trail from that run: Offering published with
`correlation=9519b52b…` → `pending_offers=1` blocks convergence →
`ingest_tephra hohi resolved offer 9519b52b… — 0 still pending` →
generated CLIPS hook asserts `edge_replied('e1')` → converged.

## Gotchas (bit us during verification)

- Vary the query between runs: Dis's toolbox evaluate cache memoizes LLM
  calls by payload, so an identical query returns the cached answer.
- Dis's ritual registry is in-memory — after a Dis restart, deactivate and
  re-activate the config or Runs fail against an orphaned ritual id.
- Focus `clara_mind_splinter` (the script does this) or the local ponder
  answers with the echo evaluator.
